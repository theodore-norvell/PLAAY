Blocks

Blocks consist of sequences of declarations and expression

The value of a block is the value of its final declaration.

Expressions

Expression place holder

You can replace this place holder with any kind of expression.

Alternation expressions (also known as if expressions)

Guard expression
First block
Second block

Shortcut key: ?

Alternation expressions are used to make decisions.

If the guard expression evaluates to true, the first block is evaluated.

If the guard expression evaluates to false, the second block is evaluated.

Note that the guard expression must be a boolean expression.

Iteration expressions (also known as while expressions)

Shortcut key: @

Iteration expressions are used to do something repeatedly.

Iteration expressions repeatedly evaluate a block of code until the guard expression is false.

Variables

Shortcut keys

Creating expressions with the keyboard

Infix/postfix example: x enter , tab 2 enter * y enter ` f enter space + 1 tab

Prefix example: , x tab tab + * 2 tab ` f tab y tab tab tab 1 tab

  • 0-9 will create a numeric literal expression starting with that digit. Works with the numpad as well.
  • a-z and A-Z will create a variable expression starting with that character.
  • ' (apostrophe) will create an empty variable expression.
  • " will create a string literal.
  • ` (back quote) will create a call expression.
  • . will create a selection expression.
  • For each of the above use tab or enter to finish.
  • +, -, *, /, <, >, %, &, |, and = will create the appropriate call expression.
  • ? will create an alternation (if) expression.
  • @ will create a while-loop expression.
  • \ will create a lambda expression.
  • _ will create an application expression
  • ( will create a tuple expression.
  • $ will create an object literal expression.
  • [ will create an index expression.
  • : will create an assignment expression.
  • ; will create a loc expression.

Creating declarations with the keyboard

Variable declaration example: a b c enter , or , a b c tab

Location variable declaration example: a b c enter , space ; or , a b c enter ;

For parameters, the , may be omitted.

  • , will create a variable declaration.
  • ; will turn plain variable declarations into location variable declarations (or vis versa) when the declaration is selected.

Creating types with the keyboard

  • " will insert the string type.
  • ? will insert the boolean type.
  • 0 will insert the natural number type.
  • 1 will insert the integer type.
  • 2-9 will insert the number type.
  • \ will insert a function type.
  • ; will insert a location type.
  • : will insert a field type.
  • ( will insert a tuple type.
  • & will insert a meet type.
  • | will insert a join type.

Moving around with the keyboard

  • ← → ↓ ↑ to change the selection.
  • shift+ ← → ↓ ↑ to expand or contract the selection.
  • space to expand the current selection outward.
  • enter to edit the text associated with the selection.
  • tab to move the selection to the next dropzone or place-holder.
  • shift+tab to move the selection to the previous insertion point.
  • ctrl+a or cmd+a to select everything.

Other editing with the keyboard

  • ctrl+c or cmd+c to add the current selection to trash.
  • ctrl+x or cmd+x to cut the current selection to trash.
  • ctrl+v or cmd+v to paste the top (most recently added) item from the trash.
  • ctrl+b or cmd+b to swap the current selection with the top of the trash.
  • crtl+0, ..., ctrl+9 or cmd+0, ..., cmd+9 to paste an item from the trash. 0 will paste the top trash item. 1 will paste the second trash item, 2 the third item, and so on.
